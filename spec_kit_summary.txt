SPEC-KIT OVERVIEW (Condensed)

1.  COS’È SPEC-KIT Spec‑kit è un framework per Spec‑Driven Development:
    le specifiche (in Markdown) sono la fonte principale da cui si
    derivano design, task e implementazione. Non è un agent framework ma
    un workflow engine per guidare un singolo coding agent.

2.  STRUTTURA GENERALE .specify/ memory/constitution.md -> regole
    globali del progetto specs// spec.md -> cosa costruire
    (requirements) plan.md -> come costruirlo (architettura) tasks.md ->
    task eseguibili scripts/ -> automazione e orchestrazione

3.  RUOLO DEI FILE MARKDOWN

-   constitution.md: system prompt persistente del progetto
-   spec.md: specifiche funzionali
-   plan.md: design tecnico e architettura
-   tasks.md: grafo di esecuzione per l’implementazione
-   AGENTS.md: istruzioni per gli agenti su come navigare e usare il
    repo

I file .md non sono solo documentazione: sono input eseguibili per
l’LLM.

4.  RUOLO DELLA CLI La CLI specify:

-   crea la struttura del progetto
-   genera template markdown
-   esegue script shell
-   collega i comandi slash agli agenti (es. /speckit.plan)

Markdown = stato e configurazione Scripts + CLI = macchina a stati che
guida il processo

5.  COME FUNZIONANO GLI “AGENTI” Spec‑kit non implementa agenti propri.
    Usa agenti esterni (Claude, Copilot, Gemini, Cursor) e li controlla
    tramite:

-   prompt template in markdown
-   file AGENTS.md
-   configurazioni interne nella CLI (AGENT_CONFIG)

Quindi il sistema è: CLI -> prompt markdown -> agente esterno -> LLM

6.  AGENTS.MD AGENTS.md è un README per gli agenti:

-   descrive struttura del progetto
-   comandi disponibili
-   convenzioni e workflow Gli agenti lo leggono automaticamente come
    contesto iniziale.

7.  AGENT CONFIG La CLI contiene una registry degli agenti supportati.
    Per ogni agente definisce:

-   cartelle dove mettere i prompt
-   formato dei comandi
-   script di setup

Per supportare un nuovo agente bisogna modificare il codice della CLI,
non basta aggiungere markdown.

8.  SKILLS E MCP Spec‑kit non ha un sistema nativo di skills o MCP.
    Esistono integrazioni esterne che espongono spec‑kit come tool MCP,
    ma non fanno parte del core.

9.  MODELLO MENTALE Spec‑kit è simile a Terraform: Markdown
    (dichiarativo) -> LLM reasoning -> codice

Non è multi‑agent: È single‑agent, multi‑phase prompting.

10. RIASSUNTO ARCHITETTURALE

User ↓ specify CLI ↓ AGENT_CONFIG + scripts ↓ Markdown templates
(constitution/spec/plan/tasks) ↓ Agente esterno (Claude/Copilot) ↓ LLM
